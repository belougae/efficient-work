# efficient-work
Happy coding easy life


1. 在做任何事之前，先定义完成的标准。
2. 在做任何需求或任务之前，先定好验收标准。
3. 尽早提交代码去集成。
4. 默认所有需求都不做，直到弄清楚为什么要做这件事。
5. 面对需求和技术难题时不要局限于工具人，程序员等角色就极有可能柳暗花明又一村。
6. 在动手做一件事之前，先推演一番。
7. 问一下自己，我的工作是不是可以用数字衡量。遇到问题最好拿数字说话。
8. 多写单元测试，代码作者比测试人员更了解代码。
9. 单元测试是重构的前提和保障。
10. static function 不利于测试所以要尽量T避免。
